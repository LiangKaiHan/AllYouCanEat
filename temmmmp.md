題名は、エラリー・クイーンの口癖で数学用語のQ.E.D.にちなむ。新シリーズ、iffの由来は同値を表す英語「if and only if」より。
---
1990年代に連載が開始された推理漫画の中では水準が高いとされている[5]。第1シリーズ単行本第17巻の初版帯で法月綸太郎が[注 3]、第2シリーズ『iff』単行本第2巻の初版帯で田中芳樹が、それぞれ本作に対する推薦文を寄せた。

2005年10月より、『月刊少年マガジン』にて連載が始まった別作品『C.M.B. 森羅博物館の事件目録』は、姉妹編にあたり、『Q.E.D.』と同一世界の出来事という設定。『C.M.B.』の第2巻に1コマだけであるが水原警部が登場している。

掲載当時の『マガジンGREAT』（2005年9月号）に掲載された『C.M.B.』の広告には「（Q.E.D.の主人公である）燈馬君のイトコが大活躍するぞ!!」の煽り文句が記された。この広告と同様のものが、『C.M.B』第1巻の広告として単行本23巻に収録されている。

第44話「ライアー」では、最後で燈馬兄妹の従弟（『C.M.B. 森羅博物館の事件目録』の主人公・榊森羅）の存在が明かされている。同作はこの新連載の告知の意味合いも兼ねていた。

『Q.E.D』第28巻と『C.M.B.』第6巻では、エジプトを舞台とした想と森羅の競演が実現。『Q.E.D』第41巻と『C.M.B.』第19巻では国際司法裁判所でそれぞれの国の代表となった想と森羅の法廷対決が描かれた。

是的，你完全不用自己「先建一個物件」！ GitHub 會自動幫你處理 Git 的 blob（物件）跟 tree（目錄結構）。你只要用手機瀏覽器（Safari 或 Chrome）操作就行，GitHub 官方 App 目前還不支援直接上傳圖片（這是很多人卡住的主因）。
我把最簡單、最推薦的兩種方法整理給你，依照你的需求選：
方法 1：最推薦（如果你正在編輯 Markdown 檔案）
這是最快的，尤其你上次在 Edit 頁面裡（就是那張有「阿仁炒麵滷雞翅」的畫面）。
	1	回到你正在編輯的那個檔案頁面（如果還在 Edit 模式就最好）。
	2	在編輯器的上方工具列，找到 小圖片圖示（通常在文字格式工具列的最右邊，或旁邊有「Attach files」）。
	3	點擊它 → 直接跳出手機的相簿 / 照片選擇器。
	4	選你要上傳的圖片（可以一次選多張）。
	5	GitHub 會自動上傳到你的 repo，並在編輯區插入 Markdown 語法： ![](https://github.com/你的帳號/你的repo/raw/main/圖片檔名.jpg)
	6	
	7	最後按 Commit changes 就完成。
優點：一次搞定上傳 + 插入文字，最省事。
方法 2：想先建立乾淨的 images/ 資料夾（你說的「建一個物件」）
如果你希望圖片統一放在 images/ 資料夾裡，方便未來管理：
	1	進入你的 repo 主頁（不是 Edit 頁面）。
	2	點右上角 Add file → Upload files。
	3	在「Commit changes」上面的檔案上傳區，點 Choose your files（或直接把圖片拖進去）。
	4	從手機相簿選圖片（Android 選 Files → Images；iOS 直接選照片）。
	5	重要：在「Commit changes」下方的「File path」或上傳前，把檔名改成 images/你的圖片.jpg（這樣就會自動建立 images 資料夾）。
	6	寫個 commit 訊息 → Commit changes。
小技巧：想先建立空資料夾，可以先用「Create new file」建立 images/.gitkeep，Commit 後再進去上傳圖片。
上傳完之後怎麼用？

	•	在 Markdown 裡貼上： ![描述文字](https://github.com/你的帳號/你的repo/raw/main/images/圖片.jpg)
	•	
	•	如果你想用更穩的 CDN 連結，可以之後再改用 https://raw.githubusercontent.com/...
注意事項（避免你上次遇到的 branch 錯誤）
	•	上傳前先下拉刷新頁面（或切換成桌面版網站：Safari 點右上「Aa」→ 要求桌面版網站）。

答案是：不需要你自己手動建物件！ GitHub 會自動幫你建立 blob（檔案物件）和 tree（目錄結構）。手機版只是 UI 比較隱藏或簡化，沒有明顯的「上傳」大按鈕而已。
為什麼你現在看不到上傳按鈕？
	•	如果你編輯的檔案不是 .md 結尾的 Markdown 檔案（看起來你的檔案是純文字列表），編輯器就不會出現 Markdown 工具列的小圖片 icon 或「Attach files」區塊。
	•	手機瀏覽器版（Safari / Chrome）的 Edit 模式，工具列會縮得很小，底部也可能沒顯示「拖曳或選擇檔案」的提示。
	•	手機版 drag & drop 本來就不好用，所以 GitHub 故意把上傳功能藏在主頁面的「Upload files」裡，而不是 Edit 畫面。
最簡單、最穩的手機上傳方法（推薦，2 分鐘搞定）
先取消這次編輯（避免又遇到上次那個 branch 錯誤）：
	1	在 Edit 畫面左上角點 X（或 Cancel）→ 回到檔案內容頁面。
	2	再下拉刷新一次頁面（確保分支是最新的）。
	3	回到倉庫主頁面（就是看到所有檔案列表的那一頁，不是單一檔案）。
	4	在檔案列表上方，點 Add file 下拉選單 → 選擇 Upload files。
	5	畫面會出現一個大區塊：
	◦	點「choose your files」（或直接把手機相簿的圖片拖進去，但手機拖不太方便，就點它）。
	◦	跳出手機相簿 → 選你要上傳的圖片（可以一次選多張）。
	6	重要：想把圖片放在資料夾裡（例如 images/ 資料夾） 在上傳區的最下方，有一個「Commit changes」上面的檔案路徑欄位（或檔名欄），把檔名改成： images/你的圖片檔名.jpg
	7	 （GitHub 就會自動幫你建立 images/ 資料夾，不用你先建！）
	8	下面寫個 commit 訊息（例如「上傳滷雞翅照片」），然後點 Commit changes。
上傳完成後，GitHub 就會自動把圖片變成 repo 裡的檔案。
上傳完後，怎麼把圖片插入你的列表檔案？
	1	點進剛剛上傳的圖片檔案。
	2	點右邊的 Raw 按鈕 → 複製整個網址（就是 https://raw.githubusercontent.com/... 那串）。
	3	回到你的列表檔案，點 Edit 重新編輯。
	4	在你想插入圖片的地方，貼上 Markdown 語法： ![滷雞翅照片](https://raw.githubusercontent.com/你的帳號/你的repo/main/images/你的圖片.jpg)
	5	 （如果沒資料夾，就把 images/ 拿掉）
其他手機小技巧（如果還是不順）
	•	開啟桌面版網站（最保險）：在 Safari / Chrome 點右上「Aa」或選單 →「要求桌面版網站」。這樣 Edit 模式會出現完整的工具列和 Attach files 按鈕。
	•	如果是 .md 檔案：在 Edit 畫面，往編輯區最下方看，有一行小小的「Attach files by dragging & dropping, selecting or pasting them.」點它就能選圖片。
	•	複製貼上法（超快）：先在手機相簿把圖片「複製」，然後回到 Edit 畫面直接長按貼上（如果有支援的話）。
